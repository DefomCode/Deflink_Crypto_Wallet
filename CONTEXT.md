# DefLink Crypto Wallet (DCW) — Контекст разработки

## Текущий статус: Этап 4 ЗАВЕРШЕН ✅

### Что реализовано (Этапы 1-4):
1. **Кошельки:** Создание (AES-256), импорт приватных ключей, удаление, переименование. Хранение в `~/.config/dcw/vault.json`.
2. **Баланс:** Проверка ETH + USD через публичные RPC (PublicNode) и CoinGecko. Graceful fallback при отсутствии сети.
3. **Транзакции:** Отправка ETH с защитой резерва (0.0003 ETH), экран подтверждения с балансом/остатком, подписание, отправка, ожидание статуса.
4. **История:** SQLite БД (`~/.config/dcw/dcw.db`), автозапись при отправке, автообновление pending→success/failed, пагинация (`--page`, `--limit`), команда `dcw tx <хеш>` для деталей и ссылки на Etherscan.
5. **UX:** Все команды гибридные (one-liner + интерактив), полный русский язык в CLI, защита от перезаписи кошельков, подтверждение удаления именем.

### Структура проекта:

```bash
crypto_cli/
├── cli.py              # Все команды Typer (create, import-key, list, balance, pay, history, tx, rename, delete)
├── core/
│   └── eth.py          # Web3: баланс, подготовка/подпись/отправка транзакций, ожидание receipt
├── storage/
│   ├── crypto_vault.py # AES-256 шифрование, CRUD vault.json
│   └── db.py           # SQLite: transactions, wallet_cache, пагинация
└── utils/
    └── api.py          # CoinGecko курс ETH/USD
```
### Ключевые константы:
- `MIN_RESERVE_ETH = 0.0003` — несгораемый резерв в `cli.py`
- `ETH_RPC_URL = "https://ethereum-rpc.publicnode.com"` — в `core/eth.py`
- БД: `~/.config/dcw/dcw.db`
- Vault: `~/.config/dcw/vault.json`
